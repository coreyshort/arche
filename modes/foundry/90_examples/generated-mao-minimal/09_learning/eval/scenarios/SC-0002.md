## Purpose
Prove a feedback entry can be turned into an actionable backlog item and (optionally) a change request.

## Steps
1) Add a new entry to `09_learning/feedback-log.md` using FB-YYYYMMDD-### format.
2) Create a matching item in `09_learning/issue-backlog.md`.
3) Draft a CR in `09_learning/change-requests.md` (optional for minimal MAO, recommended).

## Pass criteria
- Feedback entry exists
- Backlog entry exists and references the feedback ID
- (Optional) CR drafted and references the issue

## Outputs
- FB entry
- ISS entry
- (Optional) CR entry
